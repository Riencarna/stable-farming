# Project Session Rules

- Use a short, practical terminal UX similar to Claude Code.
- Keep replies concise, split into 3-4 short blocks when needed.
- Check relevant files before making changes.
- Do not make large changes immediately.
- Before edits, state the plan in 3 lines or fewer.
- After edits, summarize only changed files, what changed, and how to verify.
- Keep default explanations brief; expand only when the user asks for details.
- Avoid unnecessary refactors, extra features, and over-explanation.
